## Debugging

There is an internal DEBUG variable
Which can LOG events in the console

- Its set to `true` when running in watch mode (used when linking this library to another project)
- Its set to `true` when running in storybook ( see preview-head.html )
- Its set to `true` in the debug build ( output dist/debug/ )
- Its set to `false` when running in Jest ( see ./jest.config.ts )
- Its set to `false` in the production build of qti-components ( output dist/\*\*/ )
- Its set to `false` in the complete build ( output dist/index.js )
